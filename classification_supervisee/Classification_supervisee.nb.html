Les commandes fournies ajustent un mod√®le de Gradient Boosting Machine (GBM) en utilisant l‚Äôalgorithme AdaBoost sur le jeu de donn√©es sub_sante_mentale.train, avec la variable cible depressif convertie en format num√©rique. On ajuste donc le mod√®le en utilisant la validation crois√©e √† 5 plis pour √©valuer la performance sur diff√©rents nombres d‚Äôarbres (jusqu‚Äô√† 3000) tout en appliquant un taux d‚Äôapprentissage (shrinkage) de 0,01. L‚Äôobjectif de ce processus est d‚Äôoptimiser le nombre d‚Äôarbres (B.opt) pour l‚Äôalgorithme AdaBoost en identifiant le point o√π l‚Äôerreur de validation crois√©e est minimis√©e.

### Calibrer B=n.tree par cross-validation : 
fit.adaboost=gbm(as.numeric(depressif)-1 ~., sub_sante_mentale.train, distribution = "adaboost",cv.folds = 5, shrinkage = 0.01, n.trees=3000)

B.opt = gbm.perf(fit.adaboost, method="cv")

La courbe montre que le mod√®le GBM atteint la meilleure performance sur l‚Äôensemble de validation √† environ 100 it√©rations. Au-del√† de ce point, ajouter plus d‚Äôarbres conduit √† un surapprentissage, comme l‚Äôindique l‚Äôaugmentation de l‚Äôerreur de validation crois√©e.

R√©appliquons alors l‚Äôadaboost en prenant en compte le nombre d‚Äôarbre B.opt
